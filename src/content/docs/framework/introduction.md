The pages preceding this framework explain **why** Programmable Assurance exists.

This framework explains **how** it operates.

The Programmable Assurance Framework is the canonical operating model of the discipline. It defines the concepts, relationships, and lifecycle through which organizational intent is transformed into accountable governance decisions, continuously evaluated through evidence, and refined through feedback.

The framework is organized into four parts.

**Part I — Foundations** establishes the assumptions on which the discipline is built.

**Part II — The Operating Model** describes how governance behaves, from intent through execution and evidence to measurable outcomes.

**Part III — Continuous Assurance** introduces the mechanisms that allow governance to learn over time through governance memory, governance records, and governance economics.

**Part IV — Applying the Framework** explains how the framework relates to existing disciplines and how it can be applied regardless of technology, vendor, or implementation.

Each chapter builds upon the previous one.

The framework is intended to be read sequentially.

It is not a reference manual.

It is the architectural model of the discipline.
