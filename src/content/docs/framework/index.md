# The Programmable Assurance Framework

Governance has traditionally been treated as documentation.

Policies are written.
Controls are implemented.
Audits are performed.
Evidence is collected.

Each activity often exists inside its own silo.

Programmable Assurance begins from a different assumption.

Governance should behave like an engineering system.

Intent should continuously influence decisions.
Decisions should continuously produce evidence.
Evidence should continuously improve intent.

This framework describes the architectural principles that make that possible.
